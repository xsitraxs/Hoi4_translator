# HOI4 Localisation Translator

> Автоматический переводчик локализационных файлов Hearts of Iron IV с поддержкой множества движков, кэшированием, валидацией плейсхолдеров и постобработкой текста.

![Python](https://img.shields.io/badge/Python-3.9%2B-blue?logo=python)
![License](https://img.shields.io/badge/License-MIT-green)

---

## Что это

Инструмент с графическим интерфейсом для перевода `.yml` файлов локализации HoI4 (и других игр на движке Paradox). Работает с любой папкой `localisation/`, переводит строки через выбранный движок, кэширует результаты в SQLite и корректно обрабатывает все специфичные для игры теги.

---

## Возможности

- **Несколько движков перевода** — Google Translate (бесплатно), DeepL (API), MyMemory, заглушка для Java-сервисов
- **Автоматический фолбэк** — если основной движок недоступен, переключается на следующий по цепочке без остановки
- **Защита плейсхолдеров** — `$переменные$`, `[теги]`, `§R§W` цветовые коды, `\n`, `£иконки`, `@звуки` полностью сохраняются
- **Валидатор** — после каждого перевода проверяет что все плейсхолдеры на месте, пытается авто-фиксить потери
- **Постобработка** — исправляет типичные артефакты машинного перевода: двойные пробелы, пробелы перед запятыми, некорректные отступы после цветовых кодов
- **SQLite кэш** — уже переведённые строки не запрашиваются повторно; кэш персистентен между сессиями
- **Пропуск переведённого** — определяет уже готовые строки в выходном файле по наличию кириллицы
- **Параллельные запросы** — настраиваемый семафор (1–50), защита от бана по rate limit
- **Тёмный GUI** — интерфейс на tkinter с прогресс-баром, цветным логом и сохранением настроек

## Использование

### Через GUI

1. Запусти `python hoi4_translator.py`
2. В поле **Оригинал** укажи папку с оригинальной локализацией (`localisation/english/`)
3. В поле **Перевод** укажи куда сохранять (`localisation/russian/`)
4. Выбери движок и язык источника
5. Нажми **▶ НАЧАТЬ ПЕРЕВОД**

После завершения в папке перевода появятся `.yml` файлы с суффиксом `_l_russian`, первая строка будет автоматически заменена на `l_russian:`.

## Как работает защита плейсхолдеров

HoI4 использует специальный синтаксис внутри строк локализации. Транслятор перед отправкой в переводчик заменяет все спецтеги на нейтральные маркеры `<<PH0>>`, `<<PH1>>`, и т.д., а после перевода восстанавливает их обратно.

Защищаются:
- `$variable_name$` — скриптовые переменные
- `[GetTag]` — динамические теги
- `§Y`, `§W`, `§R`, `§!` — цветовые коды
- `\n` — переносы строк
- `£infantry` — иконки юнитов
- `@звук` — звуковые триггеры

## Валидация

После каждого перевода запускается проверка: все ли плейсхолдеры оригинала присутствуют в переводе, нет ли дублей или лишних. При обнаружении проблем:

1. Делается попытка **авто-фикса** (потерянные теги дописываются в конец строки)
2. Если авто-фикс не помог — строка помечается как **битая** и записывается в лог

В конце сессии выводится сводка: `Валидация: 847/850 OK | авто-фикс: 2 | битых: 1`

---

## Постобработка

После получения перевода автоматически применяются правила исправления типичных артефактов:

| Проблема | До | После |
|---|---|---|
| Пробел после цвет-кода | `§Y текст` | `§YТекст` |
| Пробел перед запятой | `Привет , мир` | `Привет, мир` |
| Двойной пробел | `слово  слово` | `слово слово` |
| Пробел в ёлочках | `« кавычка »` | `«кавычка»` |
| Пробелы вокруг `\n` | `текст \n текст` | `текст\nтекст` |

## FAQ

**Q: Google переводчик заблокировал — что делать?**  
A: Снизь ползунок «Параллельность» до 3–5. Программа автоматически ждёт при ошибке 429.

**Q: Можно ли переводить не в русский язык?**  
A: В текущей версии целевой язык зафиксирован как `ru`. Поддержка произвольного целевого языка запланирована.

**Q: Кэш разросся, хочу очистить**  
A: Просто удали файл `translation_cache.db` рядом со скриптом.

**Q: Программа зависает на большом моде**  
A: Нормально — перевод идёт в фоне. Смотри на прогресс-бар и строку статуса.

**Q: Почему некоторые строки не переводятся?**  
A: Строки, состоящие только из плейсхолдеров (`$val$`, `[tag]`) не отправляются переводчику — в них нечего переводить.

## Известные ограничения

- Целевой язык зафиксирован: только `→ русский`
- Нет CLI-режима (только GUI)
- `os.startfile()` для открытия папки работает только на Windows; на Linux/macOS игнорируется
- JavaSim — заглушка для тестирования, не выполняет реальный перевод
