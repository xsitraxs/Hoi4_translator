# Hoi4 Translator

> Автоматический переводчик локализационных файлов Hearts of Iron IV с поддержкой множества движков и языков.

*Automatic translator for Hearts of Iron IV localisation files. Supports multiple translation engines, any target language, real-time progress, translation preview, placeholder validation, and persistent SQLite cache.*

## Что это

Инструмент с графическим интерфейсом для перевода `.yml` файлов локализации HoI4 (и других игр на движке Paradox). Работает с любой папкой `localisation/`, переводит строки через выбранный движок, кэширует результаты в SQLite и корректно обрабатывает все специфичные для игры теги.

---

## Возможности

- **Несколько движков перевода** — Google Translate (бесплатно), DeepL (API), MyMemory
- **Любой целевой язык** — русский, украинский, польский, немецкий, китайский, японский и ещё 10+
- **Автоматический фолбэк** — если основной движок недоступен, переключается на следующий по цепочке без остановки
- **Защита плейсхолдеров** — `$переменные$`, `[теги]`, `§R§W` цветовые коды, `\n`, `£иконки`, `@звуки` полностью сохраняются
- **Валидатор** — после каждого перевода проверяет что все плейсхолдеры на месте, пытается авто-фиксить потери
- **Постобработка** — исправляет типичные артефакты машинного перевода: двойные пробелы, пробелы перед запятыми, отступы после цветовых кодов
- **Двойной прогресс-бар** — отдельно по файлам и по строкам внутри каждого файла в реальном времени
- **Предпросмотр перед записью** — таблица оригинал | перевод с возможностью отредактировать каждую строку вручную
- **SQLite кэш** — уже переведённые строки не запрашиваются повторно; кэш персистентен между сессиями
- **Параллельные запросы** — настраиваемый семафор (1–50), защита от бана по rate limit
- **Тёмный GUI** — интерфейс на tkinter с цветным логом и сохранением всех настроек

## Использование
### 1 вариант

1. Запустить exeшник
2. В поле **Оригинал** укажи папку с оригинальной локализацией (`localisation/english/`)
3. В поле **Перевод** укажи куда сохранять (`localisation/russian/`)
4. Выбери движок, язык источника и **целевой язык**
5. Нажми **▶ НАЧАТЬ ПЕРЕВОД**

### 2 вариант
1. Запусти `python hoi4_translator.py`
Все остальные пункты такие же как и в первом варианте

После завершения в папке перевода появятся `.yml` файлы с суффиксом под выбранный язык, первая строка будет автоматически заменена (например `l_russian:`).

## Предпросмотр перевода

Включается галочкой **«Предпросмотр перед записью»**. После перевода каждого файла открывается модальное окно с таблицей оригинал | перевод — каждую строку можно отредактировать прямо в поле, и только после подтверждения файл будет записан на диск.

## Как работает защита плейсхолдеров

Перед отправкой в переводчик все спецтеги заменяются на нейтральные маркеры `<<PH0>>`, `<<PH1>>` и т.д., а после перевода восстанавливаются обратно.

Защищаются: `$variable_name$`, `[GetTag]`, `§Y §W §R §!`, `\n`, `£infantry`, `@sound`.

После восстановления запускается валидатор — проверяет что все теги оригинала присутствуют в переводе, нет дублей или лишних. При проблемах делается авто-фикс, иначе строка помечается как битая и пишется в лог.

В конце сессии: `Валидация: 847/850 OK | авто-фикс: 2 | битых: 1`

## Постобработка

| Проблема | До | После |
|---|---|---|
| Пробел после цвет-кода | `§Y текст` | `§YТекст` |
| Пробел перед запятой | `Привет , мир` | `Привет, мир` |
| Двойной пробел | `слово  слово` | `слово слово` |
| Пробел в ёлочках | `« кавычка »` | `«кавычка»` |
| Пробелы вокруг `\n` | `текст \n текст` | `текст\nтекст` |

## FAQ

**Q: Google заблокировал — что делать?**  
A: Снизь ползунок «Параллельность» до 3–5. Программа автоматически ждёт при ошибке 429.

**Q: Кэш разросся, хочу очистить**  
A: Удали файл `translation_cache.db` рядом со скриптом.

**Q: Программа зависает на большом моде**  
A: Нормально — перевод идёт в фоне. Смотри на два прогресс-бара и строку статуса.

**Q: Почему некоторые строки не переводятся?**  
A: Строки, состоящие только из плейсхолдеров (`$val$`, `[tag]`), не отправляются переводчику — там нечего переводить.

**Q: Что делать с «битыми» строками в логе?**  
A: Открой `hoi4_translator.log`, найди строки с меткой `BROKEN` — там будет ключ и описание проблемы. Исправь вручную в выходном `.yml`.
